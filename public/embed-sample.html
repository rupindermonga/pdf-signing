Integration guide

1. Mint a short-lived embed token (server-side)

Your backend calls this with a signer's token and the origin you'll embed from. The embed token is single-use and expires in 30 min by default.

POST /api/v1/signers/<signer_token>/embed-session
Authorization: Bearer ds_live_...

{
  "allowedOrigin": "https://app.example.com",
  "ttlSeconds": 1800
}

Response:

{
  "ok": true,
  "embedToken": "...",
  "embedUrl": "https://sealforge/embed/sign/...",
  "expiresAt": "2026-04-15T20:00:00.000Z"
}

2. Embed the iframe

<iframe
  src="https://sealforge/embed/sign/<embedToken>"
  width="100%" height="800"
  allow="camera; microphone"
  style="border:1px solid #d0d5dd;border-radius:8px;"
></iframe>

3. Listen for signer events

SealForge posts message events to the parent frame. Always verify e.origin matches the SealForge domain before trusting the payload.

window.addEventListener('message', (e) => {
  if (e.origin !== 'https://sealforge') return;
  if (!e.data || e.data.source !== 'sealforge') return;
  console.log('[SealForge]', e.data.type, e.data.detail);

  switch (e.data.type) {
    case 'sf:loaded':     // signer opened the page
      break;
    case 'sf:signed':     // signer submitted signature (detail.completed = true if last)
      break;
    case 'sf:declined':   // signer declined (detail.reason)
      break;
    case 'sf:reassigned': // signer forwarded to another person (detail.to)
      break;
    case 'sf:completed':  // all signers done — poll /api/v1/documents/<uuid> or rely on webhook
      break;
  }
});

4. Security notes

  • allowedOrigin is pinned at token mint time — the iframe only renders if its parent is that exact origin.
  • Embed tokens are single-use. Mint a fresh one each time a customer loads the signer page.
  • Always validate e.origin in your message handler — never trust event data by itself.
  • Public embeds must be HTTPS. http://localhost is allowed for development only.
