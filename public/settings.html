Webhooks

Receive HTTPS POSTs when documents are sent, signed, completed, or cancelled. Each request is signed with HMAC-SHA256 in the X-SealForge-Signature header (format: t=<unix>, v1=<sha256> where the signed payload is <t>.<body>). Only https:// URLs are accepted — consumers should reject requests older than 5 minutes to prevent replay.

REST API Quick Reference

Base URL:
Auth header: Authorization: Bearer ds_live_…

Endpoints:
• GET /api/v1/me — current user info
• GET /api/v1/documents — list your documents
• POST /api/v1/documents — create (multipart: pdf, signers, title, optional fields, signingMode)
• GET /api/v1/documents/:uuid — document detail
• POST /api/v1/documents/:uuid/cancel — cancel pending document
• GET /api/v1/documents/:uuid/download — download signed PDF (when completed)
• GET /api/v1/templates — list your templates
